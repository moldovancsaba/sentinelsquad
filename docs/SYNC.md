# Sync

This page explains the sync relationship between issues, board state, product work, and documentation.

## Sync Model

- Issues store delivery intent and evidence.
- The board stores workflow state.
- Product repositories store implementation.
- SentinelSquad stores shared rules and shared knowledge.

## Required Sync Points

- when work starts
- when ownership changes
- when status changes
- when evidence is added
- when a shared rule changes

## Documentation Sync

- update SentinelSquad docs when the change affects multiple projects or global process
- update product docs when the change is product-local
